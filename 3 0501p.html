Введение в проектирование АИС

Автоматизированные информационные системы (АИС) представляют собой сложные комплексы программных, технических, информационных и организационных компонентов. Проектирование АИС — это процесс создания проекта, который определяет структуру, компоненты, функции и документирование будущей системы.

Классификация методов проектирования АИС

Существует несколько подходов к классификации методов проектирования АИС. Рассмотрим основные из них:

По степени охвата проектных задач

Локальные методы

Ориентированы на решение отдельных задач проектирования (например, проектирование интерфейсов или баз данных).

Интегрированные методы

Охватывают все этапы жизненного цикла АИС и обеспечивают комплексное проектирование системы.

По степени формализации

Интуитивные методы

Основаны на опыте и экспертных знаниях проектировщиков. Малая формализация процессов.

Формализованные методы

Используют строгие математические модели и формальные спецификации для проектирования.

Комбинированные методы

Сочетают формальные методы с эвристическими подходами и экспертными знаниями.

По степени использования инструментальных средств

Ручное проектирование

Все проектные операции выполняются вручную без использования специализированного ПО.

Компьютерно-поддерживаемое проектирование

Используются отдельные программные средства для автоматизации частных задач проектирования.

Автоматизированное проектирование

Применяются интегрированные CASE-средства (Computer-Aided Software Engineering), охватывающие все этапы проектирования.

Основные методологии проектирования

Методология Подход Преимущества Недостатки
Структурное проектирование Нисходящее проектирование, декомпозиция на модули Четкая структура, хорошая документированность Сложность адаптации к изменениям требований
Объектно-ориентированное проектирование Моделирование системы как совокупности взаимодействующих объектов Гибкость, повторное использование компонентов Высокие начальные затраты на обучение
Прототипное проектирование Быстрое создание упрощенных версий системы для уточнения требований Быстрая обратная связь, уточнение требований Риск неоптимальных архитектурных решений
Быстрая разработка приложений (RAD) Итеративная разработка с использованием CASE-средств Сокращение времени разработки, высокая гибкость Требует высокой дисциплины команды

Стадии и этапы создания АИС

Создание автоматизированной информационной системы представляет собой сложный процесс, который включает несколько последовательных стадий. Рассмотрим общепринятую структуру жизненного цикла АИС:

1

Предпроектная стадия

На этой стадии формируются цели и концепция будущей системы, оценивается ее целесообразность и возможная эффективность.

• Обследование объекта и обоснование необходимости создания АИС
• Формирование требований пользователя
• Разработка технико-экономического обоснования
• Разработка и утверждение технического задания на проектирование
2

Проектирование

Стадия непосредственного создания проектных решений, определяющих архитектуру системы, ее функции и основные характеристики.

• Техническое проектирование (разработка архитектуры системы)
• Разработка проектных решений по подсистемам
• Проектирование информационного обеспечения
• Проектирование программного обеспечения
• Проектирование технического обеспечения
• Разработка документации
3

Разработка

Стадия реализации проектных решений, создания компонентов системы и их интеграции.

• Программирование и настройка программных модулей
• Подготовка данных и наполнение базы данных
• Создание и адаптация программных средств
• Комплексная отладка системы
4

Ввод в действие

Стадия подготовки к эксплуатации и непосредственного запуска системы в промышленную эксплуатацию.

• Подготовка к внедрению
• Проведение опытной эксплуатации
• Обучение персонала
• Сопровождение внедрения
• Подписание акта о вводе в промышленную эксплуатацию
5

Эксплуатация и сопровождение

Стадия использования системы, ее поддержки и развития в соответствии с changing требованиями.

• Эксплуатация системы
• Локализация проблем и устранение ошибок
• Анализ функционирования и модернизация
• Поддержка пользователей

Современные тенденции в проектировании АИС

С развитием технологий подходы к проектированию АИС continuously evolving. К современным тенденциям относятся:

  • Использование agile-методологий — гибких итеративных подходов к разработке
  • Микросервисная архитектура — проектирование системы как набора loosely coupled сервисов
  • DevOps-практики — интеграция процессов разработки и эксплуатации
  • Low-code/No-code платформы — ускорение разработки через визуальное проектирование
  • Облачные технологии — проектирование систем с ориентацией на облачную инфраструктуру
  • Контейнеризация — использование Docker и Kubernetes для упрощения развертывания
