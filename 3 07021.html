МДК.07.02 Сертификация информационных систем

Резервное копирование: цели, методы, концепции, планирование, роль журнала транзакций. Виды резервных копий. Утилиты резервного копирования.

1. Введение: Место резервного копирования в системе безопасности

Резервное копирование (бэкап) — это процесс создания копии данных на носителе, предназначенном для восстановления этих данных в случае их утери или повреждения. Это последний рубеж обороны в системе информационной безопасности, обеспечивающий выполнение одного из ключевых принципов — доступности (Availability).

2. Цели резервного копирования

  • Восстановление после сбоев: Аппаратные сбои, ошибки ПО, повреждение файловых систем.
  • Защита от вредоносного ПО: Восстановление данных после атак ransomware (шифровальщиков).
  • Восстановление после человеческого фактора: Случайное удаление или изменение файлов.
  • Соблюдение нормативных требований: Архивное хранение данных по закону.
  • Долгосрочное архивирование.

3. Ключевые концепции и планирование: Стратегия 3-2-1

Золотое правило 3-2-1

  • 3 копии данных: Одна основная и две резервные.
  • 2 разных типа носителей: Например, HDD/SSD и магнитные ленты или облако.
  • 1 копия хранится вне площадки (off-site).

Ключевые метрики планирования:

  • RTO (Recovery Time Objective): Целевое время восстановления. Как быстро система должна быть возвращена в работу?
  • RPO (Recovery Point Objective): Целевая точка восстановления. Сколько данных (по времени) можно позволить себе потерять?

Пример: Если RPO = 1 час, бэкапы должны создаваться не реже чем каждый час.

4. Виды резервных копий

Вид копии Что копируется? Преимущества Недостатки
Полная (Full) Все выбранные данные. Простое и быстрое восстановление. Много времени и места.
Инкрементная (Incremental) Данные, измененные с момента последнего бэкапа. Быстрое создание, малый объем. Медленное восстановление (нужна цепочка копий).
Дифференциальная (Differential) Данные, измененные с момента последней полной копии. Восстановление быстрее, чем у инкрементной. Объем больше, чем у инкрементной.

Типичный цикл: Полная копия (в воскресенье) -> Инкрементные/дифференциальные копии (с понедельника по субботу).

5. Роль журнала транзакций (Transaction Log) в резервном копировании БД

Для СУБД (SQL Server, Oracle, PostgreSQL) журнал транзакций критически важен.

  • Назначение: Фиксирует ВСЕ изменения в базе данных.
  • Роль в бэкапе:
    1. Резервное копирование "онлайн": Без прерывания работы пользователей.
    2. Восстановление на момент сбоя (Point-in-Time Recovery): Комбинация полной копии БД + копии журнала транзакций позволяет восстановить данные на любой момент времени.

Это позволяет достичь очень малого RPO (потерять всего несколько минут или даже секунд данных).

6. Утилиты резервного копирования

Категория Примеры Описание
Встроенные в ОС tar, rsync (Linux), Windows Server Backup Базовые функции, часто бесплатны.
Специализированные коммерческие Veeam, Veritas NetBackup, Commvault, Acronis Мощные корпоративные решения. Поддержка виртуальных сред, облака.
Облачные (SaaS) Azure Backup, AWS Backup, Druva Резервное копирование напрямую в облако. Минимальная инфраструктура.

7. Сертификация информационных систем: Связь с резервным копированием

7.1. Как документ (Требования регулятора)

Процедура сертификации ИС (например, по требованиям ФСТЭК России) предъявляет прямые требования к системе резервного копирования, которые фиксируются в документах:

  • Наличие регламента резервного копирования.
  • Периодичность создания копий (для обеспечения RPO).
  • Методы хранения (включая off-site).
  • Проведение регулярных учений по восстановлению (доказательство достижимости RTO).

7.2. Как «веб-страница» (Практическая реализация)

В жизненном цикле сертифицированной системы резервное копирование — это динамический механизм:

  • Панель мониторинга: Веб-интерфейсы современных систем бэкапа показывают статус задач в реальном времени — это «веб-страница» здоровья системы восстановления.
  • Прозрачность для аудита: При проверке можно наглядно продемонстрировать работу системы бэкапа через ее интерфейс.
  • Интеграция с SIEM: События бэкапа становятся частью общей "картины безопасности" в веб-интерфейсе SIEM.

8. Заключение

Резервное копирование — это сложная, хорошо спланированная процедура, являющаяся краеугольным камнем обеспечения доступности и целостности данных. Правильный выбор методов и инструментов напрямую влияет на выполнение бизнес-требований RTO и RPO. Для сертифицированных информационных систем организация резервного копирования, соответствующая нормативным требованиям, является обязательным условием.
