Лекция по МДК.07.02

Тема: Восстановление базы данных: основные алгоритмы и этапы. Брандмауэры.

1. Восстановление базы данных: основные алгоритмы и этапы

1.1. Типы сбоев СУБД

  • Мягкие сбои (Soft Crash): Сбой ОС или аварийное завершение СУБД. Структуры данных на диске не повреждены.
  • Жесткие сбои (Hard Crash): Повреждение носителя информации. Требуется восстановление из резервной копии.
  • Ошибки приложения/пользователя: Неправильные транзакции, случайное удаление данных.

1.2. Основные алгоритмы восстановления

Протокол Write-Ahead Logging (WAL)

  • Правило 1: Любое изменение сначала записывается в журнал транзакций, и только потом — в базу данных.
  • Правило 2: Запись в журнал должна быть сброшена на диск до записи измененных данных в основную базу.
  • Цель: Гарантия, что информация о любой незавершенной транзакции сохранится в журнале после сбоя.
Алгоритм Назначение Процесс
Алгоритм отката (UNDO) Отмена изменений незавершенных транзакций СУБД просматривает журнал назад и отменяет действия транзакций, которые находились в состоянии "в процессе" на момент сбоя.
Алгоритм повтора (REDO) Повтор изменений завершенных транзакций СУБД просматривает журнал вперед и повторяет действия транзакций, которые были зафиксированы, но их изменения не попали в базу.

1.3. Этапы восстановления СУБД после сбоя

  1. Анализ (Analysis): Определение состояния транзакций на момент сбоя путем чтения журнала от последней контрольной точки.
  2. Повтор (Redo): Применение всех изменений от завершенных транзакций.
  3. Откат (Undo): Отмена всех изменений незавершенных транзакций.

Контрольная точка (Checkpoint)

— момент гарантированного сброса измененных буферов из памяти на диск для ускорения восстановления.

2. Брандмауэры (Межсетевые экраны)

2.1. Определение и назначение

Брандмауэр (Firewall) — программный или аппаратный комплекс для фильтрации сетевого трафика на основе правил безопасности.

2.2. Типы брандмауэров

Тип Уровень OSI Принцип работы Плюсы/Минусы
Пакетный фильтр Сетевой (3) Анализ IP-адресов, портов, протоколов + Высокая скорость
- Не анализирует содержимое
Stateful Inspection Транспортный (4) Отслеживание состояния соединений + "Умная" фильтрация
- Большая нагрузка
Прокси-шлюз Прикладной (7) Глубокий анализ содержимого трафика + Максимальная защита
- Низкая производительность
NGFW Все уровни (3-7) Все методы + IPS, контроль приложений + Комплексная защита
- Высокая стоимость

2.3. Концепция DMZ (Demilitarized Zone)

DMZ — изолированный сегмент сети для общедоступных сервисов (веб-сервер, почта).

Схема сети:
Интернет → Брандмауэр → DMZ → Брандмауэр → Внутренняя сеть

3. Сертификация информационных систем: Связь с темами лекции

3.1. Как документ (Требования регулятора)

При сертификации ИС (ФСТЭК России) предъявляются строгие требования:

  • Для СУБД: Обязательность журналирования, регламент восстановления, регулярные учения.
  • Для брандмауэров: Обязательное использование, настроенные правила ACL, ведение журналов событий.

3.2. Как «веб-страница» (Практическая реализация)

В современной системе технологии реализуются через динамические интерфейсы:

  • Панель управления СУБД: Веб-интерфейс для мониторинга журналов, бэкапов и восстановления.
  • Панель управления NGFW: Веб-интерфейс для просмотра топологии, сессий, атак и генерации отчетов.

При аудите специалист демонстрирует работу механизмов через эти веб-интерфейсы, доказывая их эффективность.

4. Заключение

Восстановление БД и использование брандмауэров — фундаментальные аспекты обеспечения целостности, доступности и конфиденциальности информации. Алгоритмы WAL, REDO и UNDO гарантируют согласованность данных, а брандмауэры защищают сетевой периметр. Корректная настройка и документальное оформление этих механизмов являются обязательными для сертификации ИС.
