Цель занятия

Ознакомиться с основными принципами сопровождения и обслуживания информационных систем на примере платформы 1С:Предприятие. Получить практические навыки выполнения типовых операций по поддержке и администрированию системы.

По итогам занятия студенты должны уметь выполнять базовые операции по обслуживанию типовой конфигурации 1С:Предприятие, включая обновление, резервное копирование и мониторинг работы системы.

Задачи практического занятия

  1. Изучить архитектуру платформы 1С:Предприятие
  2. Освоить основные приемы работы с конфигуратором
  3. Выполнить операции по обновлению типовой конфигурации
  4. Освоить технику создания резервных копий информационной базы
  5. Провести тестирование и исправление информационной базы
  6. Изучить основные методы мониторинга производительности системы

Практическая часть

Задание 1: Подготовка рабочего окружения

Установите учебную версию платформы 1С:Предприятие и демонстрационную базу "Управление торговлей".

Шаги выполнения:

  1. Скачайте учебную версию с официального сайта 1С
  2. Установите платформу 1С:Предприятие на компьютер
  3. Добавьте демонстрационную базу "Управление торговлей"
  4. Запустите конфигуратор и ознакомьтесь с интерфейсом

Задание 2: Обновление конфигурации

Выполните обновление типовой конфигурации до последней доступной версии.

Шаги выполнения:

  1. Откройте конфигуратор базы данных
  2. Перейдите в меню "Конфигурация" → "Обновить конфигурацию"
  3. Выберите способ обновления (например, из файла обновления)
  4. Выполните сравнение и объединение конфигураций
  5. Обновите конфигурацию базы данных
  6. Запустите систему в режиме предприятия для проверки

Важно: Перед выполнением обновления обязательно создайте резервную копию информационной базы!

Задание 3: Резервное копирование и восстановление

Освойте технику создания резервных копий и восстановления информационной базы.

Шаги выполнения:

  1. Создайте резервную копию базы данных с помощью конфигуратора
  2. Изучите альтернативные методы резервного копирования (с помощью утилит)
  3. Выполните процедуру восстановления базы из резервной копии
  4. Проверьте целостность восстановленной базы данных
Пример: Интерфейс конфигуратора 1С:Предприятие

Мониторинг и оптимизация производительности

Изучите основные техники мониторинга и оптимизации производительности системы 1С:Предприятие.

Техники мониторинга:

  • Анализ журнала регистрации
  • Мониторинг блокировок и активных соединений
  • Анализ производительности запросов к базе данных
  • Использование встроенных средств мониторинга

Итоги занятия

По результатам выполнения практического занятия студенты должны:

  • Понимать архитектуру системы 1С:Предприятие
  • Уметь выполнять обновление типовых конфигураций
  • Владеть техникой резервного копирования и восстановления
  • Уметь проводить базовый мониторинг работы системы
  • Понимать основные принципы оптимизации производительности

Полученные навыки могут быть применены для сопровождения не только систем на платформе 1С, но и других информационных систем с аналогичной архитектурой.
