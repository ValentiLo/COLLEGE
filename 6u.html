Введение в качество информационных систем

Качество информационной системы — это совокупность характеристик, определяющих ее способность удовлетворять установленные и предполагаемые потребности пользователей. Оценка качества ИС является важнейшим аспектом при проектировании, разработке и внедрении систем.

Современные модели качества ИС позволяют оценивать системы по множеству критериев, обеспечивая комплексный подход к обеспечению и улучшению качества на всех этапах жизненного цикла.

Важность качества информационных систем

Обеспечение качества информационных систем критически важно по нескольким причинам:

  • Повышение удовлетворенности пользователей
  • Снижение затрат на сопровождение и доработки
  • Увеличение срока службы системы
  • Обеспечение надежности и безопасности данных
  • Соответствие нормативным требованиям и стандартам

Баланс качества

Качество ИС должно балансировать между функциональностью, производительностью, стоимостью и временем разработки

Основные модели качества

Модель МакКолла (McCall's Quality Model)

Одна из первых и наиболее известных моделей качества ПО, разработанная в 1977 году

Модель МакКолла:

Операционные характеристики
Способность к изменениям
Адаптивность к новым environment

Критерии качества по МакКоллу:

Категория Критерии
Операционные характеристики Корректность, надежность, эффективность, целостность, удобство использования
Способность к изменениям Сопровождаемость, гибкость, тестируемость
Адаптивность Портативность, повторная используемость, интероперабельность

Модель Боэма (Boehm's Quality Model)

Иерархическая модель, разработанная Барри Боэмом в 1978 году

Модель Боэма определяет высокоуровневые характеристики качества и связывает их с более конкретными атрибутами:

Утилитарность

Надежность, эффективность, удобство использования

Сопровождаемость

Модульность, понятность, тестируемость

Портативность

Адаптируемость, независимость от устройств

Стандарт ISO 9126

Международный стандарт оценки качества программного обеспечения

Стандарт ISO 9126 определяет шесть основных характеристик качества программного обеспечения:

  1. Функциональность - способность software предоставлять функции, соответствующие требованиям
  2. Надежность - способность поддерживать определенный уровень производительности
  3. Удобство использования - легкость освоения и использования software
  4. Эффективность - отношение между уровнем производительности и объемом используемых ресурсов
  5. Сопровождаемость - легкость внесения изменений и исправлений
  6. Портативность - способность software быть перенесенным из одного environment в другое

Модель Dromey

Модель, связывающая атрибуты качества с продуктом и процессом разработки

Модель Dromey фокусируется на связи между качеством конечного продукта и качеством процесса его создания:

  • Качество продукта зависит от качества его компонентов
  • Качество компонентов зависит от процесса их создания
  • Процесс должен быть ориентирован на достижение высокого качества

Метрики качества информационных систем

Для оценки качества информационных систем используются различные метрики:

Аспект качества Метрики
Функциональность Полнота реализации требований, точность выполнения операций
Надежность Среднее время между отказами, коэффициент готовности
Производительность Время отклика, пропускная способность, utilization ресурсов
Сопровождаемость Сложность кода, коэффициент комментариев, coupling модулей
Безопасность Количество уязвимостей, время устранения угроз

Обеспечение качества на практике

Для обеспечения высокого качества информационных систем применяются различные подходы:

  • Code review - проверка кода коллегами
  • Автоматизированное тестирование - unit tests, integration tests, system tests
  • Статический анализ кода - выявление потенциальных проблем до выполнения
  • Прототипирование - создание прототипов для validation требований
  • Непрерывная интеграция - регулярное слияние и тестирование кода

Качество должно закладываться в систему на этапе проектирования, а не добавляться в процессе тестирования. Проактивный подход к качеству значительно эффективнее реактивного.
