Цель занятия

Ознакомиться с основными показателями надежности информационных систем и научиться рассчитывать их на практике. Получить навыки анализа и оценки надежности систем на основе количественных показателей.

По итогам занятия студенты должны уметь рассчитывать основные показатели надежности, анализировать их значения и делать выводы о надежности информационной системы.

Задачи практического занятия

  1. Изучить основные показатели надежности информационных систем
  2. Освоить методы расчета показателей надежности
  3. Провести расчеты для различных конфигураций системы
  4. Проанализировать полученные результаты
  5. Сравнить надежность различных архитектурных решений
  6. Сформулировать рекомендации по повышению надежности

Теоретическая часть

Основные показатели надежности

Показатель Обозначение Определение Формула
Вероятность безотказной работы P(t) Вероятность того, что система будет работать безотказно в течение времени t P(t) = e-λt
Средняя наработка на отказ MTBF Ожидаемое время между отказами системы MTBF = 1/λ
Среднее время восстановления MTTR Ожидаемое время восстановления системы после отказа MTTR = Σtвосст/n
Коэффициент готовности Kг Вероятность того, что система окажется работоспособной в произвольный момент времени Kг = MTBF/(MTBF+MTTR)
Интенсивность отказов λ Количество отказов в единицу времени λ = n/Σtраб
P(t) = e-λt = e-t/MTBF
Kг = MTBF / (MTBF + MTTR)

Надежность систем с резервированием

Для повышения надежности systems используются различные схемы резервирования:

  • Hot standby — горячее резервирование, когда резервный компонент работает параллельно с основным
  • Cold standby — холодное резервирование, когда резервный компонент включается только после отказа основного
  • Warm standby — теплое резервирование, промежуточный вариант между горячим и холодным

Итоги занятия

По результатам выполнения практического занятия студенты должны:

  • Понимать основные показатели надежности информационных систем
  • Уметь рассчитывать вероятность безотказной работы и коэффициент готовности
  • Уметь анализировать и сравнивать надежность различных конфигураций систем
  • Понимать влияние резервирования на надежность системы
  • Уметь формулировать рекомендации по повышению надежности

Полученные навыки необходимы для работы системным аналитиком, архитектором информационных систем или специалистом по качеству и надежности.
